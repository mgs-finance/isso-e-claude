Aula 11.1Prática⏱ ~20 min

Claude Cowork —
O Assistente com Memória

O Cowork é a evolução do chat: o Claude acessa seus arquivos reais, organiza pastas, cria relatórios e lembra do contexto entre sessões. Nunca mais explica o projeto do zero.

O que é o Claude Cowork

Claude Cowork é uma funcionalidade da Anthropic que transforma o Claude em um agente de desktop: ele pode ler e criar arquivos no seu computador, organizar pastas, gerar documentos, agendar tarefas e manter memória persistente entre sessões.

🖥️

Cowork vs. Chat — a diferença fundamental

Chat: você conversa, pede textos, recebe respostas. Contexto reset a cada sessão. · Cowork: o Claude age no seu sistema, acessa arquivos reais, lembra o que aconteceu ontem e executa tarefas de múltiplas etapas de forma autônoma.

O que o Cowork pode fazer

📁
Organizar arquivos e pastas
Renomeia, move, organiza arquivos por projeto, data ou categoria. Cria estruturas de pastas baseadas em padrões que você define.
📊
Criar relatórios e apresentações
Lê seus dados, gera relatórios formatados em Word/PDF, monta slides no PowerPoint e consolida informações de múltiplas fontes.
🧠
Memória persistente entre sessões
O sistema "bom dia / boa noite" — você faz um briefing do dia e o Claude retoma de onde parou. Sem perder contexto de projetos em andamento.
Bom dia, Claude. Hoje preciso continuar o relatório Q2 e preparar a reunião das 14h.
🔄
Fluxos de trabalho automáticos
Processa uma pasta de documentos, extrai informações, consolida em uma planilha e envia por e-mail — tudo em sequência, sem intervenção manual.
🌐
Pesquisa e síntese de informações
Pesquisa na web, lê artigos, consolida informações e gera um documento de inteligência competitiva — como um analista de plantão.

Tarefas e Fluxos
no Cowork

O Cowork trabalha com um ciclo de planejamento e execução. Entenda o task loop e veja como criar fluxos de trabalho que o Claude executa de ponta a ponta.

O Task Loop do Cowork

Diferente do chat, o Cowork usa um ciclo de planejar → executar → revisar → iterar. O Claude divide tarefas complexas em etapas menores, executa cada uma e te informa o progresso.

Exemplo de task loop real
# Você pede: "Pegue todos os PDFs na pasta /Contratos de 2024, extraia as datas de vencimento e valores, crie uma planilha resumindo por fornecedor e me avise quais vencem nos próximos 30 dias." # O Cowork executa autonomamente: Etapa 1: Listando arquivos em /Contratos... (23 PDFs encontrados) Etapa 2: Lendo contrato_fornA_jan2024.pdf... ✓ Etapa 3: Lendo contrato_fornB_mar2024.pdf... ✓ [...mais 21 arquivos...] Etapa 4: Criando planilha contratos_resumo.xlsx... ✓ Etapa 5: Verificando vencimentos próximos 30 dias... → 3 contratos vencem em breve: - Fornecedor Alpha: R$12.400 vence em 12 dias - Fornecedor Beta: R$8.800 vence em 18 dias - Fornecedor Gama: R$34.000 vence em 27 dias

Skills —
Instruções Reutilizáveis

Skills são pacotes de instruções que o Claude carrega automaticamente quando a tarefa combina com o contexto. Uma vez criada, a Skill funciona em qualquer lugar do ecossistema Claude.

O que são Skills

Uma Skill é um arquivo SKILL.md com instruções específicas para uma tarefa ou domínio. O Claude detecta automaticamente quando deve usar cada Skill baseado na tarefa pedida — sem você precisar lembrar de incluir contexto toda vez.

💡

Analogia prática

Pense numa Skill como um "modo especialista" que o Claude ativa automaticamente. Você criou uma Skill de "análise financeira"? Sempre que você pedir análise de números, o Claude automaticamente aplica as regras, formato e profundidade que você definiu.

Anatomia de uma Skill

SKILL.md — estrutura padrão
# [Nome da Skill] ## Quando usar esta Skill [Descreva em 1-2 frases quando o Claude deve ativar esta Skill. Exemplo: "Use quando o usuário pedir análise financeira ou revisão de DRE, fluxo de caixa ou balanço patrimonial."] ## Contexto do domínio [O que o Claude precisa saber sobre este campo específico. Termos técnicos, convenções do setor, padrões da empresa.] ## Como executar 1. [Passo 1 do processo] 2. [Passo 2 do processo] 3. [Passo 3 do processo] ## Formato de saída [Como estruturar a resposta: tabela, bullet points, relatório formatado, etc. Inclua exemplos se necessário.] ## Regras importantes - [O que SEMPRE fazer] - [O que NUNCA fazer] - [Como lidar com casos especiais] ## Arquivos de referência - /Empresa/templates/modelo_relatorio.docx - /Empresa/dados/historico_financeiro.xlsx

Claude Agents —
Autonomia 24 Horas

Agentes são instâncias do Claude que operam de forma autônoma, executam tarefas em segundo plano e coordenam com outros agentes para concluir trabalhos complexos sem intervenção humana constante.

Chat vs. Cowork vs. Agents

Evolução do modo de trabalho com Claude
CHAT (reativo) → Você pergunta → Claude responde → fim → Cada sessão começa do zero → Você está presente durante toda a execução COWORK (assistido) → Claude age no seu sistema com sua supervisão → Memória persiste entre sessões → Você acompanha o progresso e aprova etapas AGENTS (autônomo) → Claude executa fluxos completos sem você presente → Pode coordenar múltiplos sub-agentes especializados → Reporta resultados quando conclui ou encontra bloqueios → Opera 24h — inclusive enquanto você dorme

Arquitetura de um sistema de agentes

🤖

Agente Orquestrador

🔍Agente PesquisaBusca e coleta dados
📊Agente AnáliseProcessa e interpreta
✍️Agente RedaçãoGera conteúdo final
📤Agente EntregaEnvia e notifica
⚠️

Atenção: agentes requerem supervisão

Agentes autônomos são poderosos mas precisam de escopo claro, limites definidos e pontos de checagem. Nunca configure um agente para agir sem pelo menos uma etapa de aprovação humana em decisões que impactam dados, finanças ou comunicações externas.

11.5 — Criando seu primeiro agente
Aula 11.5Lab⏱ ~22 min

Criando Seu
Primeiro Agente

Passo a passo para criar um agente funcional usando Claude Code com subagentes — o agente de monitoramento de concorrentes que opera automaticamente toda semana.

Agente de Inteligência Competitiva — Completo

Python — agente com subagentes via API
import anthropic import schedule import time client = anthropic.Anthropic() # Agente 1: Pesquisador de concorrentes def agente_pesquisa(concorrente): res = client.messages.create( model="claude-sonnet-4-5", max_tokens=2000, tools=[{"type": "web_search_20250305", "name": "web_search"}], system="Você é um analista de inteligência competitiva.", messages=[{"role": "user", "content": f"Pesquise novidades de {concorrente} nos últimos 7 dias."}] ) return res.content[0].text # Agente 2: Analista estratégico def agente_analise(dados_pesquisa): res = client.messages.create( model="claude-sonnet-4-5", max_tokens=1500, system="Você é um estrategista de negócios sênior.", messages=[{"role": "user", "content": f"Com base nos dados: {dados_pesquisa}\n Identifique: oportunidades, ameaças e recomendação."}] ) return res.content[0].text # Orquestrador principal def executar_monitoramento(): concorrentes = ["Empresa A", "Empresa B"] relatorio_completo = [] for empresa in concorrentes: dados = agente_pesquisa(empresa) analise = agente_analise(dados) relatorio_completo.append({"empresa": empresa, "analise": analise}) # Salvar relatório e notificar salvar_relatorio(relatorio_completo) print("✓ Monitoramento concluído. Relatório gerado.") # Agendar para toda segunda-feira às 8h schedule.every().monday.at("08:00").do(executar_monitoramento) while True: schedule.run_pending() time.sleep(60)

Padrões de segurança para agentes

Regras de ouro para agentes autônomos
✓ SEMPRE definir escopo claro: "só leia arquivos em /dados/entrada" ✓ SEMPRE ter ponto de aprovação humana antes de ações irreversíveis ✓ SEMPRE logar cada ação com timestamp para auditoria ✓ SEMPRE definir limite de tentativas: se falhar 3x, para e avisa ✓ SEMPRE testar primeiro em ambiente de homologação ✗ NUNCA dar acesso irrestrito ao sistema de arquivos ✗ NUNCA deixar agente enviar e-mails externos sem revisão ✗ NUNCA executar em produção sem pelo menos 1 semana de testes ✗ NUNCA deixar agente com acesso a cartões de crédito ou pagamentos
