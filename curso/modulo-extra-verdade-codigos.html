5 "Códigos Secretos"
Analisados um por um

Viral no Instagram, com 2.958 curtidas. Um criador afirma que ARTIFACTS, OODA, L99, /godmode e /ghost são "códigos secretos" que transformam o Claude. Vamos abrir o capô e ver o que é real.

Aula X.1 Desmistificação ⏱ ~20 minutos

Veredito por Veredito:
o Que Realmente Funciona

Pegar um vídeo viral e testar cada afirmação na prática. O que muda o resultado de uma conversa com o Claude? O que é ilusão de placebo? O que simplesmente não existe? Aqui a resposta honesta.

Por que isso importa

Quando alguém acredita que digitar "/godmode" dá acesso a um "modo oculto" do Claude, ela está tomando a decisão errada sobre onde colocar seu tempo. Em vez de aprender a estruturar melhor um prompt — que tem impacto real — ela caça truques que não existem.

A lógica de "código secreto" é antiga: funciona em games, não funciona em modelos de linguagem. Não há backdoor. Não há palavra mágica. O que existe são padrões de comunicação que ativam capacidades reais — e esses valem a pena aprender.

⚠️

O efeito placebo é real

Se você escreve "/godmode" e depois faz uma pergunta melhor formulada do que sempre fez, a resposta vai ser melhor — mas pelo prompt melhorzinho, não pelo prefixo. O viés de confirmação faz o resto. Você vai atribuir o resultado ao "código".

Os 5 — análise completa

✓ Tem lógica ARTIFACTS Palavra de instrução, não código
O que o post diz

É um "código secreto" que transforma o Claude em máquina de produtividade e gera resultados especiais.

A verdade

Não é código. É uma palavra de instrução no prompt que direciona o Claude a gerar o resultado como um artifact — HTML, documento, código — em vez de texto simples. Funciona porque é uma instrução legível que o Claude interpreta normalmente.

→ Como realmente usar
Me dê uma ideia de trabalho de física que parece difícil mas é simples. Gere como ARTIFACT: uma página HTML interativa com o conceito visualizado, passos de execução e materiais necessários.
💡

Veredicto real

Funciona. Mas não porque é "secreto" — é só uma instrução de formato. Qualquer palavra que peça o mesmo resultado funciona igual. O artifact em si (HTML, código, documento) é o que tem valor, não a palavra.

✓ Tem lógica OODA Framework real, origem militar
O que o post diz

"OODA Como ganhar dinheiro rápido na internet?" — como se fosse um prefixo mágico.

A verdade

OODA é um framework de decisão criado pelo piloto de caça John Boyd nos anos 1950. Significa Observe, Orient, Decide, Act. Quando você coloca "OODA" no prompt, está (inconscientemente) pedindo para o Claude estruturar a resposta nesse loop. Funciona porque é um framework de raciocínio legítimo, não porque é uma palavra secreta.

→ Como realmente usar (em profundidade)
Use o framework OODA para analisar minha situação: OBSERVE: Quais dados objetivos são relevantes aqui? [descreva sua situação] ORIENT: Como esses dados se relacionam? Que padrões emergem? DECIDE: Quais opções de ação surgem dessa análise? ACT: Qual é o plano concreto com primeiros passos? Situação: [seu contexto real]
✗ Placebo L99 Não significa nada para o Claude
O que o post diz

"Quais produtos digitais mais vendem hoje? L99" — como se desbloqueasse uma resposta mais profunda.

A verdade

"L99" não é um comando reconhecido, não existe em nenhum protocolo do Claude, não tem documentação da Anthropic. O Claude processa como texto aleatório no final do prompt — que ele ignora ou tenta interpretar como parte da pergunta. Qualquer melhoria na resposta vem de outros fatores.

🚫

Teste você mesmo

Faça a mesma pergunta duas vezes — uma com "L99" e uma sem. As respostas serão equivalentes. Se houver diferença, é da variação natural de linguagem, não do prefixo.

✗ Não existe /godmode Não é um comando do Claude
O que o post diz

"/godmode Quais habilidades estão pagando mais para freelancers hoje?" — como se ativasse um modo superior.

A verdade

Comandos com "/" (slash) existem em algumas ferramentas — Discord, Slack, Claude Code. No Claude.ai (chat), não há comandos de slash. "/godmode" é tratado como texto normal. O Claude não tem modos ocultos nem levels de acesso desbloqueáveis por palavras no chat. Isso não existe na arquitetura do produto.

🔬

A origem do mito

Vem da cultura de games (God Mode = invencibilidade) + prompts de jailbreak de 2022-2023 que tentavam "desbloquear" a IA com frases especiais. Aqueles também não funcionavam como prometiam — e com modelos modernos, ainda menos.

✗ Não existe /ghost Não é um comando do Claude
O que o post diz

"/ghost Qual tipo de portfólio atrai mais clientes hoje?" — suposto modo anônimo ou especial.

A verdade

Igual ao /godmode — não existe. O Claude não tem modo ghost, não tem modo anônimo, não tem modos secretos de nenhum tipo acessíveis via prefixo de texto. A pergunta é respondida normalmente, sem qualquer diferença. O prefixo "/" é completamente ignorado no contexto de resposta.

📱

Por que esses posts viralizam

Urgência ("ninguém vai te ensinar isso"), exclusividade ("5 códigos secretos"), demonstração visual convincente (tela do celular com resposta boa). O conteúdo parece legítimo porque as respostas mostradas são boas — mas isso é seleção de cherry-picking, não efeito do código.

O que funciona de verdade

A boa notícia: você não precisa de códigos secretos. As técnicas que realmente melhoram as respostas do Claude são acessíveis, documentadas e aprendíveis. Elas são exatamente o que você aprende neste curso — e vamos aprofundar nas próximas aulas deste módulo.

O que realmente muda uma resposta
❌ NÃO muda a qualidade da resposta: → Prefixos como /godmode, /ghost, L99 → Palavras "mágicas" não documentadas → Emojis no início do prompt → Afirmar que você é especialista para "enganar" o Claude → Dizer que é urgente ou importante ✓ O QUE REALMENTE MUDA: → Definir a audiência do conteúdo ("para um CEO de PME") → Especificar o formato esperado ("como tabela comparativa") → Dar contexto real ("minha empresa tem 3 pessoas, R$50k de runway") → Pedir raciocínio em voz alta ("pense passo a passo antes de concluir") → Usar frameworks reais como estrutura (OODA, SCAMPER, 5W2H) → Definir restrições negativas ("sem jargão técnico", "sem clichês") → Iterar a partir da primeira resposta em vez de aceitar e sair

RTK — Economize até 89%
dos tokens no Claude Code

Diferente dos "códigos secretos", o RTK é uma ferramenta de código aberto, instalável, com dados mensuráveis. 10,3M de tokens economizados em 2.927 comandos — e você pode replicar isso.

Aula X.2 Ferramenta Real ⏱ ~25 minutos

RTK — A Ferramenta que
Realmente Corta Tokens

RTK (Run Token Kit) é um toolkit para Claude Code que intercepta comandos comuns — busca de arquivos, git status, leitura de diretórios — e devolve apenas o que o Claude precisa saber, descartando o ruído. O resultado: 60% a 89% menos tokens por operação.

Por que tokens custam tanto no Claude Code

No Claude Code, cada ação que o agente toma consome tokens — não só o que você digita, mas também os outputs dos comandos de shell que o agente executa. O problema: muitos comandos retornam muito mais informação do que o agente precisa.

O problema — sem RTK
// Você pede: "quais arquivos JS existem no projeto?" // O agente executa: find . -name "*.js" // Output retornado: TUDO isso vai pro contexto: ./node_modules/react/index.js ./node_modules/react/cjs/react.development.js ./node_modules/react/cjs/react.production.min.js ./node_modules/react-dom/index.js ... (847 linhas de node_modules) ./src/components/Button.js ← isso é o que importa ./src/pages/Home.js ← isso é o que importa ./src/utils/api.js ← isso é o que importa // Tokens consumidos: ~6.000 para encontrar 3 arquivos úteis // Tokens que deveriam ser consumidos: ~50
Com RTK — rtk find
// RTK intercepta o comando, exclui node_modules, // .git, build/, dist/ e outros diretórios de ruído // Retorna apenas: ./src/components/Button.js ./src/pages/Home.js ./src/utils/api.js // Tokens consumidos: ~50 // Economia: 78% nesse comando específico

Os dados reais — do print que viralizou

O print mostrado no post de Instagram não é fabricado — é o output real do comando rtk gain que mostra estatísticas de economia acumulada.

Comandos principais

Quando o RTK faz mais diferença

✅

Use RTK quando...

Projetos grandes: monorepos, projetos com node_modules pesados, repositórios com histórico longo de git. O RTK economiza mais onde há mais ruído para filtrar.

💡

Impacto no custo

Se você usa o plano Max ou API direta, cada token economizado é dinheiro. Em sessões longas de Claude Code (1+ horas de desenvolvimento), o RTK pode reduzir o consumo de contexto a ponto de permitir sessões mais longas antes de precisar resetar.

⚠️

Limitação importante

O RTK é uma ferramenta de terceiros — não é oficial da Anthropic. Antes de usar em projetos de produção, verifique a versão atual, o repositório oficial no GitHub e se há atualizações. Ferramentas da comunidade podem ter breaking changes entre versões.

OODA Loop —
Além do Post Viral

O OODA é um dos frameworks de decisão mais sofisticados já criados. Foi desenvolvido para pilotos de caça lidando com situações que mudam em frações de segundo. Aplicado ao Claude, ele muda a estrutura do raciocínio — não porque é uma palavra especial, mas porque é uma arquitetura de pensamento poderosa.

Aula X.3 Framework ⏱ ~22 minutos

OODA Loop —
Do F-86 ao Claude

John Boyd era o piloto mais temido da Força Aérea americana. Nunca perdeu um duelo aéreo. Sua obsessão era entender como decisões são tomadas em ambientes caóticos e de alta velocidade. O resultado virou um dos frameworks de decisão mais usados no mundo — em militares, negócios, esportes e agora IA.

A história real do OODA

John Boyd percebeu que na Guerra da Coreia, pilotos americanos venciam combates mesmo com aviões tecnicamente inferiores (o F-86 Sabre vs o MiG-15). O motivo: o F-86 tinha uma bolha de cockpit maior e controles mais responsivos — os pilotos conseguiam observar mais rápido, reorientar mais rápido, decidir mais rápido e agir mais rápido. Quem completasse o loop mais rápido ganhava.

Boyd formalizou isso como o OODA Loop — e depois passou décadas refinando, especialmente a fase de "Orient", que ele considerava a mais negligenciada e mais poderosa das quatro.

As 4 fases — em profundidade

O
Observe — Observar
Coletar dados brutos da situação sem filtro. O que é objetivamente verdadeiro? Que fatos existem, independente da sua interpretação? Dados, números, eventos, comportamentos observáveis.
→ "Que dados objetivos e verificáveis são relevantes aqui? Separe fatos de interpretações. Liste apenas o que é mensurável ou observável."
O
Orient — Orientar
A fase mais crítica do OODA segundo Boyd. Interpretar os dados à luz de modelos mentais, experiências, cultura, análise implícita. É onde a experiência e o conhecimento transformam dados em compreensão. É aqui que o Claude entrega mais valor.
→ "Com esses dados, o que eles significam no contexto de [setor/situação]? Que padrões emergem? Que experiências ou analogias são relevantes aqui?"
D
Decide — Decidir
Gerar hipóteses de ação a partir da orientação. Não é uma decisão final — é um repertório de opções possíveis ordenadas por prioridade, cada uma com suas consequências mapeadas.
→ "Que opções de ação surgem dessa análise? Para cada uma: qual o custo, qual o benefício, qual o risco e qual o horizonte de tempo de resultado?"
A
Act — Agir
Implementar a decisão — e ao mesmo tempo iniciar um novo ciclo de Observação. O "loop" é essencial: você age, observa o resultado da ação, reorienta e decide de novo. Boyd insistia que a velocidade do loop é mais importante do que a perfeição de cada etapa.
→ "Qual é o plano concreto com primeiros 3 passos, responsáveis, prazo e métrica de sucesso? E que novo dado vou observar depois de agir para alimentar o próximo loop?"

Prompt OODA completo para usar agora

Cole esse template e adapte para seu contexto. É o OODA aplicado de forma profunda — não como prefixo de duas letras, mas como estrutura de raciocínio completa.

Template OODA completo
Aplique o framework OODA para analisar minha situação. CONTEXTO: [Descreva sua situação aqui em detalhe] --- OBSERVE: Liste apenas dados objetivos e verificáveis sobre essa situação. Separe fatos de opiniões. O que é mensurável? ORIENT: Com esses dados, o que eles significam no contexto de [seu setor]? Que padrões você reconhece? Que experiências análogas são relevantes? Quais são meus pontos cegos prováveis? DECIDE: Liste as 3 principais hipóteses de ação. Para cada uma: benefício esperado, risco principal, custo estimado. Qual a ordem de prioridade e por quê? ACT: Para a hipótese #1: plano concreto com primeiros 3 passos. Prazo de cada passo. Métrica para saber se funcionou. O que observo depois de agir para iniciar o próximo loop?
⚡

Por que é tão mais poderoso do que só escrever "OODA"

Quando você escreve "OODA Como ganhar dinheiro?" o Claude reconhece a palavra mas não sabe como você quer estruturar. Com o template completo, você instrui cada fase explicitamente — e o Claude executa cada uma em profundidade, em vez de fazer uma resposta genérica de 4 parágrafos.

10 Técnicas que
Ninguém no Instagram
Ensina

Não têm nome dramático. Não viralizam fácil. Não cabem em 30 segundos de Reels. São técnicas que requerem entender como o Claude processa informação — e que entregam resultados que "códigos secretos" jamais vão chegar perto.

Aula X.4 Técnicas Avançadas ⏱ ~30 minutos

As 10 Técnicas
Que Ficam Fora dos Virais

Cada técnica abaixo tem: o que é, por que funciona tecnicamente, o prompt exato para usar, e o resultado esperado. Salve esse módulo — ele vai servir como referência.

As técnicas

🔗 Chain-of-Density (CoD) Avançado

Técnica de resumo iterativo criada por pesquisadores da MIT e Salesforce. Em vez de pedir um resumo direto, você pede resumos progressivamente mais densos — cada versão mantém o mesmo tamanho mas adiciona mais informação essencial. O resultado final é um resumo de altíssima densidade informacional.

→ Prompt exato
Aqui está o documento: [COLE O TEXTO] Crie um resumo usando Chain-of-Density: Versão 1 (inicial): Resumo de 5 frases do texto. Pode ser genérico. Versão 2: Reescreva com o mesmo tamanho, mas adicione 3 entidades/conceitos essenciais que faltaram. Torne mais denso sem aumentar. Versão 3: Repita — adicione mais 3 elementos essenciais ao mesmo tamanho. Versão 4: Final. Adicione os últimos elementos críticos restantes. Entregue apenas a Versão 4, precedida de uma linha explicando o que foi comprimido nas iterações.
Resumos 4-5x mais informativos no mesmo espaço. Especialmente poderoso para documentos longos onde cada frase precisa carregar máxima informação.
🪞 Espelhamento de Estilo Avançado

Em vez de descrever o tom que você quer, você mostra exemplos e pede para o Claude identificar os padrões antes de escrever. Isso cria uma análise explícita do seu estilo que é muito mais precisa do que qualquer descrição verbal que você poderia dar.

→ Prompt exato
Aqui estão 3 textos que escrevi: [TEXTO 1] --- [TEXTO 2] --- [TEXTO 3] Antes de escrever qualquer coisa, analise meu estilo: - Estrutura de frase preferida (curta, longa, mista) - Nível de formalidade (1-10) - Uso de dados vs. narrativa - Padrões de abertura e fechamento - Palavras que aparecem com frequência - O que eu evito Confirme seu entendimento do meu estilo. Depois escreva: [SEU PEDIDO AQUI] — com o meu estilo, não o de Claude.
Textos que parecem ter sido escritos por você, não pelo Claude. Elimina o problema de "parece escrito por IA".
🎯 Decomposição de Problema com Reconvergência Pro

Para problemas complexos, você explicitamente instrui o Claude a: (1) decompor o problema em sub-problemas independentes, (2) resolver cada um separadamente, (3) reconverger identificando as interações entre as soluções. Isso evita que o Claude simplifique problemas complexos em uma resposta única e plana.

→ Prompt exato
Problema: [DESCREVA SEU PROBLEMA] Estruture a análise em 3 fases: FASE 1 — Decomposição: Identifique os sub-problemas independentes que compõem o problema principal. Liste cada um com sua lógica interna. FASE 2 — Análise independente: Para cada sub-problema, desenvolva a análise como se fosse o único problema a resolver. Não misture ainda. FASE 3 — Reconvergência: Agora identifique: como as soluções de cada sub-problema interagem? Há conflitos? Há sinergias? Qual é a solução integrada que considera todas as interações?
Análises que respeitam a complexidade real do problema em vez de colapsarem tudo em uma resposta simples.
📐 Calibração de Incerteza Avançado

O Claude naturalmente confia nas suas afirmações de forma relativamente uniforme, mesmo quando a certeza varia muito entre elas. Ao pedir calibração explícita, você força distinção entre o que é bem fundamentado e o que é especulativo — e torna a resposta muito mais utilizável para decisões reais.

→ Prompt exato
Responda a [PERGUNTA/ANÁLISE]. Para cada afirmação relevante, marque explicitamente: [ALTO] — bem fundamentado, múltiplas fontes convergentes [MÉDIO] — razoável, mas com variação por contexto [BAIXO] — especulativo, baseado em lógica geral, verifique No final, liste as 3 afirmações com menor confiança e explique como verificá-las de forma independente.
Você sabe exatamente no que confiar e o que checar antes de usar. Elimina o problema de tratar tudo que o Claude diz como igual.
🔄 Prompt Recursivo — Claude Avalia o Próprio Output Pro

Em vez de iterar você mesmo sobre a resposta, você pede para o Claude avaliar o próprio output contra critérios explícitos que você definiu — e depois reescrever considerando a própria avaliação. O resultado é uma segunda versão que já incorporou autocrítica estruturada.

→ Prompt exato
PASSO 1: [SEU PEDIDO ORIGINAL AQUI] --- PASSO 2: Agora avalie o que você escreveu contra esses critérios: - Está direto ao ponto? (1-10) - Tem exemplos concretos suficientes? (1-10) - O tom está adequado para [AUDIÊNCIA]? (1-10) - Faltou algum aspecto crítico? O quê? - O que está mais fraco no texto? PASSO 3: Reescreva incorporando sua própria avaliação. Entregue apenas a versão final, não a avaliação intermediária.
Textos que já passaram por uma iteração de revisão sem você precisar fazer nada — economiza rodadas de vai-e-vem.
🗺️ Mapeamento de Território Desconhecido Avançado

Quando você entra em um assunto novo, em vez de fazer perguntas pontuais, você pede ao Claude para mapear o território completo — o que existe, o que você precisa saber, o que é mais importante, e quais são as perguntas certas a fazer. Funciona como um orientador antes de uma jornada.

→ Prompt exato
Preciso aprender sobre [TEMA] para tomar uma decisão sobre [OBJETIVO]. Mapeie o território para mim: 1. Os 5 conceitos fundamentais que preciso dominar 2. As 3 maiores armadilhas que iniciantes caem nessa área 3. O que especialistas sabem que iniciantes não sabem 4. As perguntas que eu deveria estar fazendo (e não estou) 5. Onde as fontes de maior qualidade existem sobre isso 6. O que geralmente é superestimado vs. subestimado nessa área Não responda ainda as perguntas — só mapeie o território.
Uma orientação completa antes de aprofundar, evitando que você invista tempo nas perguntas erradas.
⚡ Parallel Persona Debate Expert

Para decisões com múltiplas perspectivas legítimas, você instrui o Claude a simular um debate entre 3-4 personas com visões distintas sobre o mesmo problema — e depois sintetizar o debate. É mais rico do que pedir "prós e contras" porque as personas têm valores e prioridades diferentes, não só argumentos.

→ Prompt exato
Situação: [DESCREVA SUA DECISÃO/DILEMA] Simule um debate entre estas personas sobre o que eu deveria fazer: PERSONA A — O Pragmático: foco em resultado rápido, ROI, praticidade PERSONA B — O Estrategista: foco em posicionamento de longo prazo, vantagem competitiva PERSONA C — O Cético: questiona premissas, aponta riscos e pontos cegos PERSONA D — O Inovador: busca a solução não-óbvia, questiona o padrão do setor Rodada 1: Cada persona expõe sua posição (3-4 frases cada) Rodada 2: Cada persona responde à mais forte objeção das outras Síntese: Que elementos do debate merecem mais peso? Conclusão integrada.
Uma visão 360° de uma decisão, com tensões reais entre perspectivas — muito mais útil do que qualquer lista de prós e contras.
📊 Análise de Segundo e Terceiro Grau Pro

A maioria das análises para na consequência imediata. Análise de segundo grau pergunta: "e depois que isso acontecer, o que acontece?" Terceiro grau: "e depois disso?" Essa profundidade é onde o pensamento estratégico real acontece — e o Claude faz isso excepcionalmente bem quando instruído.

→ Prompt exato
Decisão/evento: [DESCREVA] Analise as consequências em 3 graus: 1º GRAU (consequências diretas e imediatas): O que acontece imediatamente como resultado direto? 2º GRAU (consequências das consequências): Para cada consequência de 1º grau, o que ela vai desencadear? Considere reações de: mercado, concorrentes, clientes, equipe. 3º GRAU (consequências das consequências das consequências): Para as consequências mais relevantes do 2º grau, para onde elas apontam no médio prazo (6-18 meses)? Síntese: Qual é a consequência de maior impacto que provavelmente está sendo ignorada hoje?
Visão de consequências que a maioria das pessoas só descobre depois de tomar a decisão — especialmente útil em estratégia de negócios e carreiras.
🧬 Decomposição Bayesiana de Crenças Expert

Inspirada no raciocínio bayesiano: explicitar qual a sua crença atual sobre algo, que evidência mudaria essa crença, e como novas informações devem ser ponderadas. É uma forma de tomar decisões sob incerteza de forma estruturada — e de usar o Claude para identificar quais evidências realmente importam buscar.

→ Prompt exato
Crença atual: "[DECLARE SUA CRENÇA OU HIPÓTESE AQUI]" Minha confiança nessa crença: [X]% Análise bayesiana: PRIOR: Por que acredito nisso? Quais experiências e dados me levaram aqui? LIKELIHOOD: Que evidências confirmariam essa crença? Que evidências a negariam? Liste as 3 mais importantes de cada tipo. ATUALIZAÇÃO: Se as evidências negativas que listei forem verdadeiras, minha confiança deveria cair para [X]%. Por quê? Se as positivas forem verdadeiras, deveria subir para [Y]%. Por quê? PRÓXIMO PASSO: Que UMA informação eu deveria buscar primeiro para atualizar minha crença da forma mais eficiente possível?
Clareza sobre o que você sabe, o que está assumindo, e onde colocar energia de pesquisa para reduzir incerteza de forma eficiente.
🏗️ Scaffolding — Construção Incremental de Raciocínio Pro

Em vez de dar todo o problema de uma vez, você constrói o raciocínio incrementalmente em várias mensagens — cada uma confirmando e expandindo a anterior. Isso evita que o Claude "resolva" o problema prematuramente antes de ter todo o contexto, e produz análises muito mais alinhadas com a complexidade real da sua situação.

→ Prompt exato (sequência de 3 mensagens)
// MENSAGEM 1 — só o contexto, sem a pergunta "Vou te dar contexto sobre minha situação antes de fazer uma pergunta. [CONTEXTE SUA SITUAÇÃO]. Não responda ainda. Só me diga se entendeu e se precisa de mais informação." // MENSAGEM 2 — adicionar restrições e variáveis "Certo. Agora adicione: [RESTRIÇÕES, OBJETIVOS, LIMITAÇÕES]. Ainda não responda. Confirme que entendeu todas as variáveis." // MENSAGEM 3 — finalmente, a pergunta real "Com todo esse contexto, [SUA PERGUNTA REAL]. Leve em consideração tudo que discutimos, especialmente [PONTO CRÍTICO]."
Respostas calibradas para o contexto completo, não para uma versão simplificada do problema. Especialmente poderoso para análises estratégicas longas.

O padrão comum dessas 10 técnicas

Todas elas compartilham uma característica: estruturam o processo de raciocínio, não só o resultado. Em vez de pedir "me dê a resposta", elas instruem como chegar à resposta. É essa diferença — processo vs. resultado — que separa o uso avançado do uso básico do Claude.

🔬 Quiz — Módulo Extra
1. Qual desses "códigos" do post viral tem uma lógica técnica real por trás, mesmo que o post não explique corretamente?
✅ Resposta: c. OODA é um framework real criado por John Boyd — Observe, Orient, Decide, Act. Quando você usa OODA no prompt, está invocando uma estrutura de raciocínio legítima. O Claude a conhece e a aplica. L99, /godmode e /ghost não têm nenhum significado técnico no contexto do Claude.
2. O RTK (visto no primeiro post) economizou 89,2% dos tokens em uma sessão real. O que explica esse resultado?
✅ Resposta: b. O RTK intercepta comandos como find e git status e filtra o output antes de enviar ao contexto do Claude. Em vez de mandar 847 linhas de node_modules para o agente analisar, manda só os 3 arquivos relevantes. É filtragem de ruído — simples e eficaz.
3. Você usa a técnica de Calibração de Incerteza. Qual é o benefício principal?
✅ Resposta: c. A calibração de incerteza não muda o que o Claude sabe — muda como ele comunica a confiança em cada afirmação. Você recebe uma resposta onde sabe distinguir o que é bem fundamentado do que é especulativo, e sabe o que checar. O Claude ainda pode confabular — mas agora você tem o mapa para identificar onde.
4. Por que a técnica de Scaffolding (construção incremental) produz resultados melhores do que enviar tudo de uma vez?
✅ Resposta: b. Quando você dá todo o contexto e a pergunta de uma vez, o Claude já começa a processar a resposta enquanto ainda "lê" as últimas partes. O scaffolding garante que cada pedaço do contexto seja confirmado antes de prosseguir — a análise final tem o contexto completo incorporado, não parcial.
5. Um post viral afirma que digitar "ULTRAMODE" antes de qualquer pergunta faz o Claude "usar todo seu potencial". Qual é a análise correta?
✅ Resposta: b. "ULTRAMODE" não existe em nenhum protocolo do Claude. O Claude não tem modos ocultos acessíveis por prefixo de texto. O que acontece: quem escreve o prefixo geralmente presta mais atenção ao prompt que escreve depois — e aí atribui a melhoria ao "código". É viés de confirmação clássico. O Claude processa maiúsculas e minúsculas de forma equivalente para instruções desse tipo.
