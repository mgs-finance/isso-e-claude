Aula 07.1Prática⏱ ~18 min

Instalação e Configuração
do Claude Code

Claude Code é a ferramenta de linha de comando da Anthropic para desenvolvimento assistido por IA. Você trabalha no terminal, no seu projeto real, com acesso total ao código.

⚠️

Versão e disponibilidade

Claude Code é atualizado frequentemente pela Anthropic. Os comandos e comportamentos descritos neste módulo refletem a versão disponível na data de criação do curso. Verifique a documentação atual em docs.anthropic.com/claude-code para novidades. Requer plano Pro, Max ou Team.

O que é o Claude Code

Claude Code é um agente de codificação que roda no terminal. Diferente de um chat, ele tem acesso direto aos seus arquivos, pode editar código, rodar comandos, criar arquivos e navegar pelo seu projeto — tudo com sua supervisão.

⚡

O que o Claude Code pode fazer

Ler e editar qualquer arquivo do projeto · Executar comandos no terminal · Criar e deletar arquivos · Entender toda a base de código · Refatorar, debugar, testar e documentar · Criar features completas a partir de uma descrição

Comandos Essenciais

/help
Lista todos os comandos disponíveis e atalhos

Limpa o histórico da sessão atual (novo contexto)
/diff
Mostra todas as mudanças feitas na sessão

Modo Agentic —
Autonomia Real

O modo agentic permite que o Claude execute múltiplas etapas de forma autônoma, sem confirmar cada passo. Poderoso — mas requer cuidados específicos.

Interativo vs Agentic

Comparação de modos
MODO INTERATIVO (padrão) → Claude pergunta antes de cada mudança significativa → Você aprova ou rejeita cada etapa → Mais seguro · Mais controle · Mais lento → Ideal para: código crítico, projetos novos, aprendizado MODO AGENTIC (--dangerously-skip-permissions) → Claude executa tudo sem pedir confirmação → Mais rápido · Completa tarefas complexas em autonomia → Requer: git limpo, diretório isolado, confiança no escopo → Risco: pode modificar/deletar arquivos inesperadamente

Quando usar o modo agentic

⚠️

Regra de ouro antes de modo agentic

Sempre faça git commit antes de uma sessão agentic. Se algo der errado, você volta com git checkout . Sem commit, não há volta.

Modo agentic — uso correto
# Sempre commite antes! $ git add . && git commit -m "checkpoint antes do claude" # Inicie em modo agentic $ claude --dangerously-skip-permissions > Crie um módulo completo de relatórios PDF: > - leia dados do banco SQLite em db/sales.db > - gere relatório mensal com gráfico de vendas > - salve em reports/YYYY-MM.pdf > - adicione testes unitários para a geração Claude: Executando em modo autônomo... ✓ Analisando db/sales.db (12 tabelas encontradas) ✓ Criando reports/__init__.py ✓ Criando reports/monthly.py ✓ Instalando dependências: reportlab, matplotlib ✓ Criando tests/test_reports.py ✓ Rodando testes: 8/8 passando → Módulo de relatórios criado com sucesso!

Casos de uso ideais para o modo agentic

O que delegar com segurança
✓ Criar módulos novos do zero (não toca código existente) ✓ Escrever documentação e comentários ✓ Criar testes para código existente ✓ Migração de dados entre formatos ✓ Scaffolding inicial de projetos ✓ Refactoring com escopo bem definido ✗ Modificar banco de dados de produção ✗ Deletar arquivos sem especificar quais ✗ Qualquer coisa em sistemas sem backup ✗ Código de segurança crítico sem revisão

CLAUDE.md —
Memória do Projeto

O arquivo CLAUDE.md é o "cérebro" do seu projeto para o Claude Code. Ele lê automaticamente toda vez que você inicia uma sessão — sem precisar re-explicar o contexto.

O que é o CLAUDE.md

Um arquivo Markdown na raiz do projeto (ou subpastas) que o Claude Code lê automaticamente. É como um onboarding permanente: você explica o projeto uma vez e o Claude sempre começa com esse contexto.

📋

Hierarquia de leitura

Claude lê: CLAUDE.md na raiz → CLAUDE.md em subpastas relevantes → Arquivo de contexto específico informado por você. Você pode ter um CLAUDE.md global em ~/CLAUDE.md com suas preferências pessoais.

Template Completo de CLAUDE.md

📄CLAUDE.md — template copiável
# CLAUDE.md — [Nome do Projeto] ## Visão Geral [Uma frase descrevendo o que o projeto faz] ## Stack Técnica - Backend: Python 3.11 + FastAPI - Banco: PostgreSQL 15 (ORM: SQLAlchemy) - Frontend: React 18 + TypeScript - Testes: pytest + Jest ## Estrutura de Diretórios ``` meu-projeto/ ├── app/ # Backend FastAPI │ ├── models/ # Modelos do banco │ ├── routes/ # Endpoints da API │ └── services/ # Lógica de negócio ├── frontend/ # React app └── tests/ # Testes automatizados ``` ## Convenções de Código - Nomes em inglês, comentários em português - Funções: snake_case · Classes: PascalCase - Sempre adicionar type hints em Python - Commits no formato: feat/fix/refactor: descrição ## Comandos Importantes ```bash make dev # Inicia ambiente de desenvolvimento make test # Roda todos os testes make migrate # Aplica migrações do banco ``` ## Regras para o Claude Code - Sempre rode os testes antes de declarar tarefa concluída - Nunca modifique migrations existentes — crie novas - Variáveis de ambiente ficam em .env (nunca commitar) - Ao criar endpoints, sempre adicionar validação Pydantic

MCP — Conectando
Ferramentas ao Claude Code

MCP (Model Context Protocol) é o protocolo que permite ao Claude Code se conectar a ferramentas externas — bancos de dados, APIs, navegadores, e muito mais.

O que é MCP

MCP é um protocolo aberto criado pela Anthropic. Ele permite que servidores externos exponham ferramentas e dados que o Claude pode usar. Com MCP, o Claude Code pode navegar na web, consultar bancos, operar o Slack, e muito mais.

🔌

Servidores MCP populares

filesystem (acesso a arquivos) · github (repositórios e PRs) · sqlite (consultas SQL diretas) · puppeteer (automação de browser) · slack (enviar/ler mensagens) · google-maps · postgres · notion · e centenas mais no registro oficial

Configuração MCP no Claude Code

~/.claude/config.json
{ "mcpServers": { "filesystem": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-filesystem", "/home/user/projetos"] }, "github": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-github"], "env": { "GITHUB_TOKEN": "ghp_seu_token" } }, "sqlite": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-sqlite", "--db-path", "./database.db"] }, "puppeteer": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-puppeteer"] } } }

Veja uma sessão completa criando um sistema para gerenciar manutenções de motos, com banco de dados SQLite, API e relatórios.

Sessão real — sistema de frota
# Passo 1: Definição do escopo > Crie um sistema Python de controle de manutenção de frota. > Requisitos: > - Banco SQLite com tabelas: motos, manutenções, alertas > - CLI simples para registrar e consultar manutenções > - Alerta automático quando km de revisão se aproxima > - Relatório de custos por moto no mês Claude: Vou criar o sistema em etapas. Padrões que tornam uma sessão eficiente

Estratégia de sessão ideal
1. COMECE GRANDE, REFINE PEQUENO Dê o escopo completo no início, depois peça ajustes pontuais Não fique mudando de ideia no meio — o contexto fica confuso 2. SEMPRE PEÇA PARA TESTAR "Rode os testes" ou "teste com dados reais" após cada feature Claude identificará e corrigirá erros automaticamente 3. REVISE O /diff ANTES DE COMMITAR Veja tudo que foi mudado antes de aceitar Use /diff para entender cada alteração 4. SESSÕES FOCADAS 1 objetivo por sessão (ex: "adicionar auth JWT") Sessões longas = contexto confuso = qualidade piora Use /compact se a sessão ficar muito longa 5. COMMIT FREQUENTE git commit após cada feature que funciona Nunca perca trabalho por sessão sem checkpoint

Boas Práticas
e Segurança

Claude Code tem acesso ao seu sistema. Algumas práticas simples eliminam a maioria dos riscos e tornam seu fluxo de trabalho muito mais produtivo.

Tabela de Segurança — Faça e Não Faça

SituaçãoFaçaNão Faça
Antes de modo agenticgit commitRodar sem checkpoint
Arquivos sensíveisAdicionar ao .gitignore + .claudeignoreDeixar acessível sem restrição
Banco de produçãoUsar banco de desenvolvimento/stagingApontar direto para produção
Código de segurançaRevisar manualmente antes de commitarAceitar sem revisar
API keys no códigoUsar variáveis de ambiente (.env)Deixar no código-fonte
Sessão longa/compact a cada 30-40 trocasContinuar sem compactar
Feature críticaPedir revisão passo a passoModo agentic sem supervisão
