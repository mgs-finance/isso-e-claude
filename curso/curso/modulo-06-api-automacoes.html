Aula 06.1Prática⏱ ~22 min

Sua Primeira Chamada de API

Sair do chat e falar diretamente com o Claude via código abre um mundo de possibilidades. Aqui você faz sua primeira chamada funcional em Python e JavaScript.

Por que usar a API?

O Claude.ai é ótimo para uso interativo. Mas quando você quer integrar o Claude nos seus sistemas, automatizar tarefas repetitivas, criar produtos ou processar centenas de documentos, a API é o caminho.

🔑

O que a API permite que o chat não permite

Processar documentos em lote · Integrar com seus sistemas · Criar produtos com IA embutida · Controlar cada detalhe do comportamento · Escalar sem limites manuais · Automatizar fluxos completos

Passo 1 — Obter sua API Key

1
Acesse console.anthropic.com
Crie conta ou faça login. É separado do Claude.ai — você vai precisar de créditos de API.
2
Vá em "API Keys" → "Create Key"
Dê um nome descritivo. A chave começa com sk-ant- e aparece apenas uma vez — salve em lugar seguro.
3
Configure como variável de ambiente
Nunca coloque a chave no código diretamente.
export ANTHROPIC_API_KEY="sua-chave" (Linux/Mac)

Primeira chamada — Python e JavaScript

Python
# pip install anthropic import anthropic client = anthropic.Anthropic() # lê ANTHROPIC_API_KEY do ambiente message = client.messages.create( model="claude-sonnet-4-5", max_tokens=1024, messages=[ {"role": "user", "content": "Explique API em 2 frases."} ] ) print(message.content[0].text)
JavaScript / Node.js
// npm install @anthropic-ai/sdk import Anthropic from '@anthropic-ai/sdk'; const client = new Anthropic(); // lê process.env.ANTHROPIC_API_KEY const message = await client.messages.create({ model: 'claude-sonnet-4-5', max_tokens: 1024, messages: [{ role: 'user', content: 'Explique API em 2 frases.' }] }); console.log(message.content[0].text);

Entendendo a resposta

JSON — objeto de resposta
{ "content": [{ "type": "text", "text": "Uma API é..." }], "model": "claude-sonnet-4-5", "stop_reason": "end_turn", "usage": { "input_tokens": 25, // tokens que você enviou "output_tokens": 89 // tokens da resposta (= custo) } }
💡

Controlando o custo

O custo é calculado por tokens: input + output. Use max_tokens para limitar respostas. Use system prompts curtos. Monitore o campo usage para rastrear consumo.

System Prompts via API

O parâmetro system é onde você define identidade, regras e contexto permanente do assistente. Aqui você transforma o Claude num especialista customizado.

System vs User — a diferença que importa

No chat você escreve tudo na mesma caixa. Na API há separação clara: system define o contexto permanente (quem é, como se comporta) e user é o que o usuário digita em tempo real.

Python — com system prompt
message = client.messages.create( model="claude-sonnet-4-5", max_tokens=1024, system="""Você é o assistente de cobrança da Moto Express. Resolva pendências de forma cordial. Formate valores como R$ X.XXX,XX. Nunca prometa descontos sem autorização. Responda em português, tom profissional e acolhedor.""", messages=[ {"role": "user", "content": "Quero negociar minha dívida de R$ 850"} ] )

Template de System Prompt para API

Template reutilizável com XML
SYSTEM_PROMPT = """ <identidade> Você é Assistente Comercial, especialista em atendimento ao cliente. Trabalha para Minha Empresa e atende clientes B2B. </identidade> <tom> [profissional / amigável / técnico / consultivo] Idioma: português brasileiro </tom> <regras> - [O que deve fazer] - [O que NÃO deve fazer] - [Como lidar com casos especiais] </regras> <contexto> [Informações sobre produtos, preços, políticas, FAQ] </contexto> """

Conversas Multi-turn via API

Python — mantendo histórico
# A API é stateless — você mantém o histórico history = [] def chat(user_msg): history.append({"role": "user", "content": user_msg}) res = client.messages.create( model="claude-sonnet-4-5", max_tokens=1024, system=SYSTEM_PROMPT, messages=history ) reply = res.content[0].text history.append({"role": "assistant", "content": reply}) return reply chat("Qual o prazo de entrega?") chat("E para fora de SP?") # contexto mantido!
⚠️

Atenção ao custo do histórico

Cada chamada envia todo o histórico. Em sessões longas, o custo cresce exponencialmente. Mantenha apenas as últimas N mensagens ou resuma o histórico antigo.

06.3 — Automação com Make.com

Automação Sem Código
com Make.com

Sem escrever uma linha de código, conecte o Claude a Gmail, WhatsApp, Notion, Sheets e centenas de outros apps com Make.com.

Como funciona o Make.com

Make.com é uma plataforma visual de automação. Você cria "cenários" conectando módulos em sequência: quando X acontece, faça Y com Claude, depois Z. Sem código.

⚡

Casos de uso populares com Claude + Make

Novo e-mail → Claude classifica e responde · Formulário preenchido → Claude gera proposta · Produto novo no Shopify → Claude escreve descrição e posta · Mensagem WhatsApp → Claude cria ticket de suporte

Passo a Passo — Primeiro Cenário

1
Crie conta em make.com
Plano gratuito tem 1.000 operações/mês — suficiente para testar. Clique em "Create a new scenario".
2
Adicione um Trigger (gatilho)
Ex: Gmail → Watch Emails (novo e-mail), Sheets → Watch Rows (nova linha), Webhook (chamada HTTP).
3
Adicione módulo HTTP → Make a Request
Aqui você chama a API do Claude. Configure URL, método POST e o JSON body.
Não há módulo nativo Claude ainda — use HTTP Request
4
Configure o JSON Body com variáveis
Use variáveis dinâmicas do Make (ex: {{1.body}}) para personalizar o prompt com dados do trigger.
5
Adicione ação de saída
Use a resposta do Claude: enviar e-mail, salvar no Sheets, postar no Slack, criar tarefa no Asana.

Configuração do Módulo HTTP no Make

HTTP Request — configuração completa
URL: https://api.anthropic.com/v1/messages Method: POST Headers: x-api-key: SUA_API_KEY anthropic-version: 2023-06-01 content-type: application/json Body (JSON): { "model": "claude-sonnet-4-5", "max_tokens": 500, "system": "Você é um assistente de atendimento...", "messages": [{ "role": "user", "content": "{{1.body}}" }] } Parse response: Yes (JSON) Acesse resposta: {{http.content[].text}}

n8n — Automação Self-Hosted

O n8n é como o Make.com, mas você instala no seu servidor. Sem limite de operações, dados ficam com você, e tem nó nativo do Claude.

Make.com vs n8n

Comparação
Make.com ✓ Zero configuração — começa em minutos ✓ Interface polida · Suporte incluído ✗ Limite de operações · Dados passam pelos servidores deles ✗ Custo escala com volume n8n self-hosted ✓ Ilimitado — sem custo por operação ✓ Dados no seu servidor · Open source ✓ Nó nativo de Claude/AI ✗ Requer servidor (VPS ~R$40/mês) ✗ Você gerencia a infraestrutura

Instalando o n8n

docker run -it --rm \ --name n8n -p 5678:5678 \ -v ~/.n8n:/home/node/.n8n \ -e N8N_BASIC_AUTH_ACTIVE=true \ -e N8N_BASIC_AUTH_USER=admin \ -e N8N_BASIC_AUTH_PASSWORD=senha123 \ docker.n8n.io/n8nio/n8n # Acesse: http://localhost:5678
npm install n8n -g n8n start # Acesse: http://localhost:5678
# Hospedagem gerenciada — sem instalação # Acesse: n8n.io/cloud # Trial gratuito de 14 dias · Plano ~$20/mês

Workflow Exemplo — Resumo de E-mail

Fluxo n8n
Nó 1: Gmail Trigger → Evento: "Message Received" → Filtro: label = "Para Resumir" Nó 2: Anthropic (Claude) → Model: claude-sonnet-4-5 → System: "Você é um assistente executivo..." → Prompt: "Resuma em 3 bullets: {{$json.body}}" Nó 3: Gmail → Reply → Para: {{$node["Gmail Trigger"].json["from"]}} → Corpo: "Resumo automático:\n{{$node["Claude"].json.text}}"

Google Sheets com IA Nativa

Crie uma função =CLAUDE() no Google Sheets com Apps Script. Qualquer pessoa da equipe usa IA diretamente na planilha, sem saber programar.

O que você vai criar

Uma função customizada que permite escrever =CLAUDE("Classifique: "&A2) e ter o Claude respondendo diretamente na célula.

🧮

Casos de uso reais

=CLAUDE("Classifique como Positivo/Negativo: "&A2) · =CLAUDE("Traduza para inglês: "&B3) · =CLAUDE("Extraia o CNPJ desta string: "&C5) · =CLAUDE("Esta descrição está completa? Responda Sim ou Não: "&D2)

Código Apps Script Completo

Google Apps Script
// Extensões → Apps Script → cole → salve → recarregue a planilha const ANTHROPIC_API_KEY = PropertiesService .getScriptProperties().getProperty('ANTHROPIC_API_KEY'); /** * Chama o Claude de uma célula do Sheets * @param {string} prompt O prompt para o Claude * @param {string} [system] System prompt opcional * @returns {string} Resposta do Claude * @customfunction */ function CLAUDE(prompt, system = 'Responda de forma concisa.') { const payload = { model: 'claude-haiku-4-5-20251001', // Haiku = mais barato para volume max_tokens: 256, system: system, messages: [{ role: 'user', content: String(prompt) }] }; const options = { method: 'post', contentType: 'application/json', headers: { 'x-api-key': ANTHROPIC_API_KEY, 'anthropic-version': '2023-06-01' }, payload: JSON.stringify(payload) }; const res = UrlFetchApp.fetch('https://api.anthropic.com/v1/messages', options); return JSON.parse(res.getContentText()).content[0].text; } // Configure a API key: // Arquivo → Propriedades do projeto → Adicionar propriedade: ANTHROPIC_API_KEY
💡

Dicas de performance

Use Haiku para tarefas simples (classificar, traduzir, extrair) · Limite max_tokens ao mínimo · Teste em 5 linhas antes de rodar em 500 · Nunca deixe fórmulas em colunas com recálculo automático

Streaming — Respostas
em Tempo Real

Em vez de esperar a resposta completa, o streaming mostra o texto chegando token por token. Essencial para qualquer produto com boa UX.

Por que Streaming importa

Sem streaming, o usuário fica olhando para uma tela vazia por 5-15 segundos. Com streaming, ele começa a ler imediatamente. A diferença na percepção de velocidade é enorme — e a implementação é simples.

Streaming em Python e Flask (SSE)

Python
with client.messages.stream( model="claude-sonnet-4-5", max_tokens=1024, messages=[{"role": "user", "content": "Escreva um relatório..."}] ) as stream: for text in stream.text_stream: print(text, end="", flush=True) final = stream.get_final_message()
Flask — Server-Sent Events
from flask import Flask, Response, request import anthropic app = Flask(__name__) client = anthropic.Anthropic() @app.route('/stream', methods=['POST']) def stream_response(): user_msg = request.json['message'] def generate(): with client.messages.stream( model="claude-sonnet-4-5", max_tokens=1024, messages=[{"role": "user", "content": user_msg}] ) as stream: for text in stream.text_stream: yield f"data: {text}\n\n" yield "data: [DONE]\n\n" return Response(generate(), mimetype='text/event-stream')
