Os 4 tipos de produto com IA

💬
Chatbot Especialista
IA treinada para um domínio específico — atendimento, suporte, consultor de nicho.
Ex: chatbot jurídico, assistente de cobrança, consultor de nutrição
⚡
Automação de Processo
IA que substitui etapas manuais repetitivas — classifica, transforma, gera.
Ex: classificador de leads, gerador de contratos, resumidor de e-mails
📄
Ferramenta de Conteúdo
IA que gera conteúdo personalizado em escala — posts, relatórios, descrições.
Ex: gerador de posts, criador de descrições de produtos, blog automático
🔍
Analisador de Dados
IA que lê documentos, extrai informações e gera insights estruturados.
Ex: extrator de contratos, analisador de NF, resumidor de relatórios

Chatbot Especialista
em 1 Hora

Vá do zero a um chatbot funcional com interface web em 60 minutos. Um arquivo HTML + API do Claude — sem framework, sem servidor, sem banco de dados.

O que você vai construir

Um chatbot especialista completo: interface de chat profissional, histórico de conversa, streaming de respostas, system prompt configurável e botão de limpar. Pronto para usar com clientes.

⚡

Arquitetura simplificada

1 arquivo HTML (interface + lógica) + API do Claude. Sem servidor — o usuário precisa ter a API key, ou você cria uma camada simples de backend para não expor a chave. Para MVP: HTML puro funciona.

08.3 — RAG básico sem banco vetorial
Aula 08.3Técnica⏱ ~22 min

RAG Básico Sem
Banco Vetorial

RAG (Retrieval Augmented Generation) permite que o Claude responda com base nos seus documentos. Para começar, você não precisa de banco vetorial — Python puro resolve.

O que é RAG e por que importa

RAG é a técnica de buscar informação relevante nos seus documentos e injetá-la no contexto antes de chamar o LLM. Resultado: respostas baseadas nos seus dados, atualizados, sem fine-tuning.

🔍

RAG simples vs RAG com banco vetorial

Simples (Python puro): busca por palavras-chave, funciona para até ~50 documentos, zero infraestrutura. Com banco vetorial (Pinecone, ChromaDB): busca semântica, escala para milhões de documentos, requer setup.

RAG em Python — Busca por Palavras-Chave

Python — RAG simples
import anthropic import os from pathlib import Path client = anthropic.Anthropic() # 1. Carregue seus documentos def load_docs(folder="./docs"): docs = {} for f in Path(folder).glob("*.txt"): docs[f.name] = f.read_text(encoding="utf-8") return docs # 2. Busca simples por palavras-chave def retrieve(query, docs, top_k=3): query_words = set(query.lower().split()) scores = [] for name, content in docs.items(): words = set(content.lower().split()) score = len(query_words & words) # interseção scores.append((score, name, content)) scores.sort(reverse=True) return scores[:top_k] # 3. Pergunte com contexto dos documentos def ask(question): docs = load_docs() relevant = retrieve(question, docs) context = "\n\n---\n\n".join([ f"[{name}]:\n{content[:2000]}" for _, name, content in relevant if _ > 0 ]) prompt = f"""Com base nos documentos abaixo, responda: {question} <documentos> {context} </documentos> Se a resposta não estiver nos documentos, diga isso claramente.""" res = client.messages.create( model="claude-sonnet-4-5", max_tokens=1024, messages=[{"role": "user", "content": prompt}] ) return res.content[0].text # Uso: # Coloque arquivos .txt em ./docs/ answer = ask("Qual é a política de devolução?") print(answer)
💡

Quando migrar para banco vetorial?

Use RAG simples até ter mais de 50 documentos ou perceber que a busca por palavras-chave perde resultados relevantes. Nesse ponto, ChromaDB (gratuito, local) é o próximo passo natural.

Plataformas para vender no Brasil

HOTMART (hotmart.com) ✓ Líder no Brasil para produtos digitais ✓ Aceita cartão, boleto, Pix ✓ Afiliados: você dá comissão e eles vendem por você ✓ Checkout otimizado, área de membros incluída → Taxa: 9.9% + R$1 por transação → Ideal para: cursos, mentorias, acesso a ferramentas KIWIFY (kiwify.com.br) ✓ Interface mais moderna, checkout rápido ✓ Taxa menor (5,99% + R$0,50) ✓ Integração com WhatsApp e automações → Ideal para: SaaS simples, assinaturas mensais STRIPE (stripe.com) — para SaaS ✓ Assinaturas recorrentes com controle total ✓ APIs robustas para integrar com seu produto → Taxa: 2.9% + $0.30 por transação → Ideal para: produtos técnicos com assinatura

Estratégia de Preço por Valor

💰

Princípio fundamental

Não precifique pelo custo de API. Precifique pelo valor que você entrega. Se o produto economiza 5 horas por semana de um profissional que ganha R$100/hora, ele vale R$2.000/mês — não R$50. O custo de API é seu custo, não o preço do cliente.
